# 포트폴리오 백테스터

설정형 포트폴리오 백테스터 — 적립·리밸런싱 규칙 DSL, 한국 세제 엔진, 다중 전략 비교.
**완전 독립 실행형 앱**입니다 (백엔드·로그인 없음, 데이터는 브라우저가 Yahoo Finance에서 직접 조회).

스펙 문서: [`docs/PRD.md`](docs/PRD.md) (v1.1) — 코드의 섹션 주석(§3, §4.2, 5.2 …)은 이 문서를 가리킵니다.

## 빠른 시작

```bash
npm install
npm run dev      # http://localhost:5173 (dev 서버가 /yf → Yahoo 프록시)
npm test         # 엔진 검증 스위트 (골든마스터·손계산 대조·프로퍼티, 58 tests)
npm run build    # tsc + vite 프로덕션 빌드
npm run deploy   # Cloudflare Pages 배포 (functions/yf가 프로덕션 프록시)
```

## 무엇이 다른가 (vs Portfolio Visualizer)

1. **리밸런싱 규칙 DSL** — 적립 배분(미달 우선/비례/고정 분할) × 트리거(밴드/주기/혼합) × 매도 정책(허용/무매도/무매도+주기 매도만) 조합
2. **한국 세제 엔진** — 양도세 연 손익통산 − 250만 공제 → 22% (거래별 아님), 배당 15% 원천 + 종합과세 임계 근사, 가상자산 유예 플래그
3. **거래비용·슬리피지** — 수수료 + 스프레드 bps 매 거래 반영
4. **N개 전략 동시 비교** — 같은 데이터·동일 납입 스케줄·동일 비용/세금 가정, 규칙만 다르게 (TWRR 기준 정렬)

## 아키텍처

```
[src/core]  데이터 → 엔진 → 세금 → 지표  (순수 TS, React 무관 — 전부 테스트됨)
[src/ui]    설정 편집 · 결과 시각화       (React + Recharts + Tailwind)
[functions] /yf/* Yahoo 프록시            (Cloudflare Pages Function, CORS 우회)
```

| `src/core/engine.ts` | 결정론 상태기계 — t종가 결정→t+1시가 체결, 배분/리밸런싱/체결, 리컨실 | §4 |
| `src/core/tax.ts` | 이동평균 원가·연 손익통산·배당 원천/종합과세·가상자산 플러그인 | §5 |
| `src/core/metrics.ts` | TWRR/MWRR(XIRR), growth-of-$1 기준 MDD·수면하, 연도별 서브기간 | §6 |

## 핵심 설계 결정 (PRD §14 오픈 퀘스천 확정)

| 질문 | 결정 | 근거 |
|---|---|---|
| 원가법 | **이동평균** (FIFO는 P2) | 한국 증권사 기본 (5.1) |
| 체결 타이밍 | **t 종가 결정 → t+1 시가 체결** | 룩어헤드 구조적 차단 (4.2) — 코드상 주문이 다음 루프에서만 체결됨 |
| 분수주 | **기본 허용, 토글 제공** | 정수주 모드는 잔여현금 이월 (4.5) |
| `to_underweight` 타이브레이크 | **부족분 비례 배분** (워터폴 아님) | 결정론 고정 (4.3) |
| 현금 수익 | **연율 % 설정값** (기본 4%, SGOV 근사) | 실제 경과일수 복리 (4.5) |
| 데이터 소스 | **Yahoo Finance 무료** (v8 chart API) | 비조정 close + adjclose + 배당 이벤트 동시 제공 |
| 세율 | **단일 가정 한계세율** | 정밀 누진은 P2, 근사임을 UI에 명시 (5.3) |
| 세금 납부 | **연말 현금 차감, 부족 시 다음날 비례 강제 매도** | 강제 매도 실현익은 다음 연도로 통산 (5.2) |

## 검증 전략 (§9)

- **골든마스터**: 상수 가격 60/40 → 최종가치 = 납입 합 (현금 보존) · 일 1% 성장 → 정확 복리
- **손계산 대조**: 밴드 리밸런싱 1회(매도 32주·실현익 $3,200), 연말 세금($154), 배당 원천(15%)
- **프로퍼티**: 무매도 모드 매도 0건 · 슬리브 합=포트 가치 · 현금≥0(세금일 예외) · 같은 입력=같은 출력
- **스키마 계약**: Yahoo v8 응답 형태 → 파싱 → 정렬 → 엔진 → 지표 전 파이프라인

## v1 한계 (§12 — 결과 화면에도 상시 노출)

- **USD 전용** — 원화 환산 손익(실제 양도세 기준) 아님. FX는 P2
- 세금은 가정 환율·단일 한계세율 근사 · 가상자산 과세는 유예 중(기본 비활성)
- EOD 가격 · 슬리피지 사이즈 무관 고정 bps · 시장충격 없음
- 단일 경로 백테스트 — 몬테카를로/최적화는 P2

## 이어서 작업 (P2 후보)

원화·FX 왜곡 반영(거래일 환율 실현손익) → 파라미터 민감도 스윕(밴드 5% vs 10% 자동 비교) → 롤링 윈도우 분석 → FIFO 원가법 옵션 → 몬테카를로. 착수 전 `docs/PRD.md`와 `src/core/__tests__/`를 먼저 읽으면 컨텍스트가 잡힙니다.
